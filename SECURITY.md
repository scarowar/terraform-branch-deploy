# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

Please update to the latest release before reporting a vulnerability unless the issue prevents upgrading.

## Reporting a Vulnerability

Report vulnerabilities through a [private security advisory](https://github.com/scarowar/terraform-branch-deploy/security/advisories).

Include:

- Affected version or commit SHA.
- Minimal workflow and `.tf-branch-deploy.yml` needed to reproduce the issue.
- The pull request command involved, if any.
- Expected behavior, actual behavior, and security impact.
- Relevant logs with secrets removed.

Do not report security issues in public issues, pull requests, or discussions until the issue has been reviewed and disclosure is coordinated.

## Response

We will review the report, confirm whether it affects the project, and coordinate a fix and disclosure timeline when needed. Critical issues that affect plan/apply safety, command authorization, or secret exposure are prioritized.
